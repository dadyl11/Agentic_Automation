Modul 5 — Operacje na plikach i folderach
Zadanie 1 — Kopiowanie plikow
Temat: Operacje na plikach + For Each File in Folder + Path Exists

Opis zadania

  1. Skopiuj plik Produkty.xlsx do folderu wybranego przez uzytkownika
  2. Sprawdz czy skopiowany plik istnieje w folderze docelowym
  3. Wyswietl informacje w logu: ktory plik jest procesowany i czy kopiowanie sie powiodlo

Wskazowki

  • Do pobrania biezacego folderu uzyj Get Environment Variable (zmienna: CurrentDirectory)
  • Do wyboru folderu docelowego uzyj Browse For Folder (Select Folder)
  • Uzyj For Each File in Folder z filtrem *.xlsx
  • Do kopiowania uzyj Copy File
  • Do sprawdzenia istnienia pliku uzyj Path Exists (PathType: File)
  • Sciezke docelowa buduj przez Path.Combine(str_docelowy_folder, "Produkty.xlsx")

Modul 8 — State Machine (wprowadzenie)
Zadanie — State Machine z Input Dialog
Temat: State Machine + Transitions + warunki + Final State

Opis zadania

  1. Stworz robota opartego na State Machine
  2. W stanie poczatkowym (Init) pobierz od uzytkownika imie (Input Dialog)
  3. Jesli imie to "Adam" — przypisz nazwisko "Nowak" i przejdz do Final State
  4. Jesli imie to "Kamil" — przypisz nazwisko "Kowalski" i przejdz do Final State
  5. Dla innego imienia — wrocz do stanu Init (powtorz pytanie)
  6. W Final State wyswietl Log Message: "Twoja osoba to: " + str_imie

Wskazowki

  • Stworz nowy projekt typu State Machine (nie Sequence/Flowchart)
  • Dodaj dwa stany: State (Init) i Final State
  • W stanie Init: dodaj Input Dialog w sekcji Entry
  • Stworz trzy Transitions ze stanu Init:
    • Transition 1: Warunek str_imie = "Adam" → Action: Assign str_nazwisko = "Nowak" → Final State
    • Transition 2: Warunek str_imie = "Kamil" → Action: Assign str_nazwisko = "Kowalski" → Final State
    • Transition 3: Bez warunku (domyslna) → wraca do Init
  • Final State: dodaj Log Message w sekcji Entry
Klucz: Transitions sa ewaluowane w kolejnosci — domyslna (bez warunku) musi byc ostatnia, inaczej zawsze sie wykona!

Modul 10 — Rozbudowane State Machine
Zadanie — State Machine z obsluga wyjatkow
Temat: State Machine + Try-Catch + Business/System Exception + Robotivity

Opis zadania

  1. Uzywajac State Machine stworz robota, ktory:
    • Otworzy aplikacje Robotivity (w przypadku bledu robot ma zakonczyc dzialanie)
    • Wprowadzi transakcje z pliku Transactions.xlsx
    • Na koncu wyswietli Log Message o zakonczeniu pracy
  2. Obsluz Business Exception — jesli dane nie sa numeryczne, pomin transakcje i przejdz do nastepnej
  3. Obsluz System Exception — w przypadku bledu systemowego, restartuj aplikacje i wznow prace

Stany State Machine

StanOpis
InitZbierz zmienne, wyczysc srodowisko (Kill Process), otworz aplikacje, pobierz transakcje (Read Range), Try-Catch
Get TransactionSprawdz czy jest dostepna transakcja (wiersz z DT). Jesli brak → Final State. Korzystaj ze zmiennej TransactionNumber (default = 1)
ProcessPrzetwarzaj transakcje. Business Exception → pomin, System Exception → restart. Po tranzycji wyczysc zmienna Business Rule Exception
Final StateZamknij aplikacje, wyswietl podsumowanie

Wskazowki

  • Uzyj zmiennej TransactionNumber (Int32, default: 1) do sledzenia biezacej transakcji
  • Pobieraj wiersz: dt_transakcje.Rows(TransactionNumber - 1)
  • Warunek braku transakcji: TransactionNumber > dt_transakcje.Rows.Count
  • W stanie Process: Try-Catch — Catch BusinessRuleException (pomin) i System.Exception (restart)
  • Po kazdej transakcji inkrementuj: TransactionNumber = TransactionNumber + 1

Modul 11 — REFramework
Cwiczenie wprowadzajace — REFramework
Temat: Omowienie struktury Robotic Enterprise Framework (wspolnie z prowadzacym)

Opis cwiczenia

  1. Zapoznaj sie ze struktura szablonu REFramework (Robotic Enterprise Framework)
  2. Przeanalizuj pliki w folderach: Framework/, Data/, Tests/
  3. Omow role poszczegolnych plikow:
    PlikRola
    Main.xamlGlowna State Machine (Init, Get Transaction, Process, End Process)
    InitAllSettings.xamlWczytuje konfiguracje z Config.xlsx
    InitAllApplications.xamlOtwiera wymagane aplikacje
    GetTransactionData.xamlPobiera kolejna transakcje (z Queue lub pliku)
    Process.xamlPrzetwarza pojedyncza transakcje
    SetTransactionStatus.xamlUstawia status transakcji (Success/Business/System)
    CloseAllApplications.xamlZamyka aplikacje
    KillAllProcesses.xamlAwaryjne zamkniecie procesow
    RetryCurrentTransaction.xamlLogika ponawiania transakcji
    TakeScreenshot.xamlScreenshot przy bledzie
    Data/Config.xlsxKonfiguracja: sciezki, nazwy assetow, kolejki, ustawienia
    Data/Input/Dane wejsciowe
    Data/Output/Dane wyjsciowe
    Data/Temp/Pliki tymczasowe
    Tests/Testy jednostkowe

Kluczowe pojecia

  • Business Rule Exception — blad danych (np. brak NIP) — transakcja jest pomijana
  • System Exception — blad techniczny (np. crash aplikacji) — robot restartuje aplikacje i powtarza transakcje
  • Config.xlsx — centralne miejsce konfiguracji (nie hardcoduj wartosci w workflow!)
  • MaxRetryNumber — ile razy robot powtorzy transakcje po System Exception

Config.xlsx — trzy arkusze

ArkuszOpisPrzyklad
SettingsUstawienia procesu (key-value)OrchestratorQueueName, MaxRetryNumber
ConstantsStale wartosciURL aplikacji, timeout
AssetsNazwy assetow z OrchestratoraCredentialName, LogLevel
Uwaga techniczna: Config w REFramework jest typu Dictionary(Of String, Object) — dlatego przy odczycie zawsze potrzebny jest .ToString: Config("OrchestratorQueueName").ToString

Alternatywa bez Orchestratora — Windows Credential Manager

  1. Panel sterowania → Menedzer poswiadczen → Poswiadczenia systemu Windows → Dodaj
  2. W UiPath: Get Secure Credential zamiast Get Credential
    • Target: nazwa poswiadczenia z Credential Manager
    • Output: Username (String), Password (SecureString)
Uwaga: Windows Credential Manager dziala tylko lokalnie. W produkcji zawsze uzywamy Orchestrator Assets.
